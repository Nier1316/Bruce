## 目的

验证“决策/测试任务 -> chassis_cmd 话题 -> Chassis_task 解算 -> Djimotor_set_target 静态缓冲 -> Motor_task 聚合 CAN 下发”的完整链路，并观察电机响应与底盘运动方向是否一致。

## 前置条件

- 电机接线正确，CAN1/2 速率与 ID 配置与 `dji_motor.c` 保持一致（0x1FF/0x200/0x2FF 聚合帧）。
- 底盘电机在底盘模块完成初始化并填充 `chassis_motors[4]`。
- FreeRTOS 工程正常启动，`Chassis_control_task` 和 `Motor_control_task` 已创建并运行。
- 调试串口 `debug_uart` 已初始化（用于打印测试日志）。

## 启用测试任务

- 文件：`4-Application_Layer/For_Testing_Task/chassis_motor_integration_test.c|h`
- 在 `4-Application_Layer/Robot_Task/robot_task.c` 中取消以下注释以创建测试任务：

	// osThreadDef(chassis_motor_integration_test, Chassis_motor_integration_test_task, osPriorityNormal, 0, 2048);
	// chassis_motor_integration_test_handle = osThreadCreate(osThread(chassis_motor_integration_test), NULL);

编译、下载固件后，串口应输出 [TEST]/[FBK] 日志。

## 测试步骤（默认参数）

测试任务以 100Hz 发布 `chassis_cmd`，分 6 个阶段：

1) Idle 2s：vx=0, vy=0, wz=0（确认静止）
2) 直行 3s：vx=+0.2, vy=0, wz=0（观察前进方向轮转速为正）
3) 横移 3s：vx=0, vy=+0.2, wz=0（观察横移方向发动）
4) 自转 3s：vx=0, vy=0, wz=+0.5（观察小陀螺）
5) 复合 3s：vx=+0.2, vy=+0.2, wz=0（合成斜向）
6) 停止：vx=vy=wz=0 持续发布（确认能稳停）

可根据底盘物理量单位（m/s 与 rad/s 或者归一化）适当放大小值，建议从小到大逐步验证。

## 观测与判定

- 串口日志：
	- [TEST] Phase 切换提示与参数。
	- [FBK] chassis_wz（当前示例为指令代填，后续接陀螺仪后应接近实测）。
- 物理现象：各阶段运动与设定方向一致，无异常抖动或噪音。
- CAN 报文（可选）：
	- 抓取 0x1FF/0x200/0x2FF 帧，8 字节电流值应随阶段变化。

## 常见问题与排查

- 无运动/报文：
	- 检查 Motor_task 是否运行、`Djimotor_control_all()` 是否被周期调用。
	- 确认 `DJI_Motor_Init()` 成功，电机实例已加入 `motor_instances`。
	- 检查 CAN 速率/接线/终端电阻。
- 方向错误：
	- 检查底盘运动学方向约定（坐标系、轮序），以及 `chassis_motors[i]` 与实物轮位是否一致。
- 抖动严重：
	- 调小目标量；检查 PID 参数与场景模式；确认反馈源设置（电机/外部）。

## 安全注意

- 台架首测，确保车体离地或限位，避免跌落与碰撞。
- 逐步增大目标，观察温升，避免长时间堵转。

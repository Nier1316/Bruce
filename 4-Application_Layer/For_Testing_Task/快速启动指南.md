# 底盘电机集成测试 - 快速启动指南

## 快速开始

### 1. 启用测试
在 `robot_task.c` 中选择要运行的测试：

```c
void Robot_task_init(void)
{
    // 注释掉其他测试，启用集成测试
    osThreadDef(chassis_motor_integration_test, Chassis_motor_integration_test_task, osPriorityNormal, 0, 2048);
    chassis_motor_integration_test_handle = osThreadCreate(osThread(chassis_motor_integration_test), NULL);
}
```

### 2. 硬件检查
- ✅ CAN1总线接线正确
- ✅ 4个底盘电机连接 (ID: 0x201-0x204)  
- ✅ UART1调试串口连接
- ✅ 电机供电正常

### 3. 编译下载
- 编译工程
- 下载到开发板
- 打开串口调试助手 (115200波特率)

### 4. 观察输出
正常启动会看到：
```
[CHASSIS-MOTOR-TEST] Chassis-Motor Integration Test Started
[CHASSIS-MOTOR-TEST] ✓ Chassis and Motor tasks created successfully
[CHASSIS-MOTOR-TEST] === Phase 1: Initialization Test ===
```

## 测试阶段说明

| 阶段 | 名称 | 测试内容 | 持续时间 |
|------|------|----------|----------|
| 1 | 初始化测试 | 消息中心、任务创建 | 1次 |
| 2 | 静态值测试 | 固定指令验证 | 4次循环 |
| 3 | 动态运动 | 正弦波运动 | 20次循环 |
| 4 | 模式切换 | 4种底盘模式 | 4次循环 |
| 5 | 运动学验证 | 解算正确性 | 3次循环 |

## 预期现象

### 电机运动
- **前进测试**: 前后电机转动，左右电机静止
- **横移测试**: 左右电机转动，前后电机静止  
- **旋转测试**: 四个电机同方向转动
- **复合运动**: 四个电机都有转动

## 常见问题

### ❌ 任务创建失败
```
✗ Failed to create tasks
```
**解决**: 检查堆栈空间，可能需要增加到2048字节

### ❌ 消息中心注册失败  
```
✗ Message center registration failed
```
**解决**: 确保消息中心模块正常编译链接

### ❌ 电机无响应
```
✗ All motor speeds are zero
```
**解决**: 
1. 检查CAN接线
2. 确认电机ID配置 (0x201-0x204)
3. 验证电机供电

### ❌ 数据传递异常
```
✗ No motor speed update available
```
**解决**: 检查motor_chassis_interface模块，可能是接口数据同步问题

## 调试技巧

### 1. 单步调试
可在关键函数设置断点：
- `Chassis_handle_command()` - 检查指令接收
- `Motor_control_handle_chassis_motors()` - 检查数据传递
- `Chassis_kinematics_solve()` - 检查解算过程

### 2. 日志增强
在测试代码中添加更多日志：
```c
TEST_LOG("Debug: vx=%.2f, vy=%.2f, expected_speeds=[%.1f,%.1f,%.1f,%.1f]", 
         vx, vy, speeds[0], speeds[1], speeds[2], speeds[3]);
```

### 3. 频率调整
如需更慢的测试节奏：
```c
// 将测试间隔从500ms改为2000ms
osDelay(2000);
```

## 测试通过标准

每个阶段应该显示：
```
✓ Phase X test passed
```

如果看到：
```
✗ Phase X test failed
```

请根据具体错误信息进行故障排除。

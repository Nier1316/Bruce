# 消息中心发布订阅模型测试说明

## 测试任务与结果分析

`Message_test_task` 是一个专门用于测试发布订阅模型功能的FreeRTOS任务，通过串口输出详细的测试结果。

在没有使用静态sub和pub之前，我发现运行到 test12 round 之后会出现无法为每一个测试任务新建任何pub和sub了，怀疑是内存泄漏导致的。
- 内存泄漏：每次测试都重复注册新的发布者和订阅者，但从未释放内存
- 订阅者累积：同一话题的订阅者越来越多（Test 12显示已有12个订阅者）
- 内存耗尽：STM32系统堆内存有限，最终导致malloc失败

修复方案：
- 全局实例管理：创建全局发布者和订阅者变量，只初始化一次
- 实例复用：所有测试使用相同的全局实例，避免重复注册
- 内存稳定性：系统启动后内存使用量保持稳定


## 测试覆盖的场景

### 1. 测试1：一发布者 → 一订阅者
- **目的：** 验证基本的发布订阅功能
- **测试内容：**
  - 创建一个发布者和一个订阅者
  - 发布传感器数据（包含计数器、温度、状态）
  - 验证订阅者能正确接收数据
  - 验证数据完整性和一致性

### 2. 测试2：多发布者 → 一订阅者
- **目的：** 验证多个发布者共享同一话题的功能
- **测试内容：**
  - 创建两个发布者注册相同话题
  - 验证两个发布者返回同一实例
  - 分别通过两个发布者发布不同的电机数据
  - 验证订阅者能接收到来自不同发布者的消息

### 3. 测试3：多发布者 → 多订阅者
- **目的：** 验证复杂的多对多发布订阅场景
- **测试内容：**
  - 创建两个发布者和三个订阅者
  - 通过不同发布者发布测试消息
  - 验证所有订阅者都能接收到消息
  - 验证消息内容的正确性

### 4. 测试4：一发布者 → 多订阅者
- **目的：** 验证广播功能
- **测试内容：**
  - 创建一个发布者和四个订阅者
  - 发布广播消息
  - 验证所有订阅者都接收到相同的数据
  - 验证数据一致性

### 5. 测试5：并发访问测试
- **目的：** 验证多线程环境下的数据安全性
- **测试内容：**
  - 创建多个不同话题的发布者和订阅者
  - 快速连续发布消息模拟并发场景
  - 验证不同话题间不会相互干扰
  - 验证数据完整性

## 测试数据结构

### sensor_data_t
```c
typedef struct {
    uint32_t counter;    // 计数器
    float temperature;   // 温度值
    uint8_t status;      // 状态标志
} sensor_data_t;
```

### motor_data_t
```c
typedef struct {
    uint16_t motor_id;   // 电机ID
    int16_t speed;       // 电机速度
    int16_t current;     // 电机电流
} motor_data_t;
```

### test_message_t
```c
typedef struct {
    uint8_t test_id;     // 测试ID
    uint32_t timestamp;  // 时间戳
    char message[16];    // 消息内容
} test_message_t;
```

## 使用方法

### 1. 添加到项目
将测试任务文件添加到项目的测试模块中：
```
4-Application_Layer/For_Testing_Task/
├── message_test_task.h
└── message_test_task.c
```

### 2. 在主任务中创建
```c
// 在 main.c 或任务管理文件中添加
osThreadDef(message_test, Message_test_task, osPriorityNormal, 0, 256);
osThreadCreate(osThread(message_test), NULL);
```

### 3. 配置串口输出
确保 `huart1` 已正确配置并可用于调试输出。

### 4. 运行测试
- 系统启动后，测试任务会自动开始
- 每轮测试间隔5秒
- 测试结果通过串口输出

## 故障排查

### 常见问题

1. **注册失败**
   - 检查内存分配是否成功
   - 确认FreeRTOS互斥锁创建正常

2. **数据验证失败**
   - 检查数据结构定义是否一致
   - 验证内存对齐问题

3. **串口输出异常**
   - 确认串口配置正确
   - 检查波特率设置

### 调试建议

1. 可以通过修改测试间隔时间来调整测试频率
2. 可以单独启用某个测试场景进行调试
3. 增加更多的错误检查和状态输出

## 注意事项

1. 测试任务会持续运行，适合长期稳定性测试
2. 每次测试都会创建新的发布者和订阅者实例
3. 由于 `QUEUE_SIZE=1`，新消息会覆盖旧消息
4. 测试结果包含成功标记（✓）和失败标记（✗）
5. 建议在相对空闲的系统状态下运行测试以获得准确结果
